# 06. Онтология сортамента

## Зачем

Сортамент — это справочник типоразмеров металлопроката. У каждого типоразмера есть параметры, которые нужны параметрическому генератору геометрии (размеры сечения, радиус гиба, толщина стенки) и экспортёрам (марка стали, ГОСТ, масса погонного метра).

Хранить эту информацию внутри DC-атрибутов или в коде плагина — неверно. Надо **отдельный набор JSON-файлов** в папке `catalog/`, которые:
- плагин читает при запуске и при выборе компонента
- пользователь может править / дополнять без перекомпиляции
- имеют понятную схему и версию
- покрывают начальный минимум, расширяются по мере работы

## Машинные типы профилей

Единый вокабулярий, используемый в `profile_type` во всех JSON, в атрибуте `nn_fabkit.profile_type` на компоненте, в коде генератора. Enum-строки, без неймспейсов.

| Ключ | Описание |
|---|---|
| `rect_tube` | Прямоугольная профильная труба (все ГОСТы: 30245, 8645, 25577) |
| `square_tube` | Квадратная труба. На практике частный случай `rect_tube`, но иногда удобно различать. |
| `round_tube` | Круглая электросварная или бесшовная труба |
| `angle_equal` | Уголок равнополочный |
| `angle_unequal` | Уголок неравнополочный |
| `channel` | Швеллер (П, У, С — уточнение в params) |
| `ibeam` | Двутавр (Б, Ш, К, Д — уточнение в params) |
| `sheet` | Лист (горячекатаный, холоднокатаный, рифлёный — уточнение в params) |
| `plate` | Толстолистовой прокат (от 4 мм и выше) |
| `rebar` | Арматура |
| `round_bar` | Круг (пруток) |
| `square_bar` | Квадрат (пруток) |
| `hex_bar` | Шестигранник |

## Правила выбора радиуса гиба

Ключевой параметр для корректного NC-экспорта, про который неправильно думают «от балды». Для ГОСТ 30245 установлено:

- при толщине стенки `t ≤ 6 мм`: внешний радиус `R = 1.5 × t`
- при `t > 6 мм`: внешний радиус `R = 2.0 × t`

Это **эмпирическое правило**, которое даёт точное значение большую часть времени, но конкретные таблицы в стандарте иногда отличаются на десятые миллиметра. Для нашего MVP:
- сначала читаем `outer_radius_mm` из JSON-записи, если задан — используем его (это «ручной override»)
- если не задан — вычисляем по правилу выше
- при расхождении с измерением на реальной трубе заказчика — правим JSON-запись

```ruby
module NN::FabKit::Catalog
  def self.outer_radius_mm(profile_type, params)
    return params["outer_radius_mm"].to_f if params["outer_radius_mm"]
    t = params["wall_mm"].to_f
    t <= 6 ? 1.5 * t : 2.0 * t
  end
end
```

## Как пользоваться из генератора

```ruby
module NN::FabKit::Catalog
  CATALOG_DIR = File.join(NN::FabKit::PLUGIN_ROOT, "catalog")

  @profiles = nil

  def self.load!
    @profiles = Hash.new { |h, k| h[k] = [] }
    Dir.glob(File.join(CATALOG_DIR, "gost-*.json")).each do |path|
      data = JSON.parse(File.read(path, mode: "r:UTF-8"))
      type = data["profile_type"]
      data["items"].each do |item|
        item["_source"] = data                 # ссылка на документ
        @profiles[type] << item
      end
    end
  end

  def self.find(profile_type, typesize)
    load! if @profiles.nil?
    @profiles[profile_type].find { |it| it["typesize"] == typesize }
  end

  def self.available(profile_type)
    load! if @profiles.nil?
    @profiles[profile_type].map { |it| it["typesize"] }.sort
  end
end

# Использование
item = NN::FabKit::Catalog.find("rect_tube", "40x20x2")
item["params"]["outer_radius_mm"]   # → 3
item["derived"]["mass_per_m_kg"]    # → 1.74
```

## Канонические `typesize` — правила записи

Единообразие строк — критично (иначе поиск по typesize станет хаосом).

Правила:
- **Без пробелов**, разделитель `x` (латинская строчная x), без умножения `×` и без пробелов вокруг.
- Для прямоугольной трубы: `<big>x<small>x<wall>`. Пример: `40x20x2`.
- Для квадратной: `<side>x<side>x<wall>`. Пример: `40x40x2` (не `40x2`).
- Для круглой трубы: `<OD>x<wall>`. Пример: `89x3`.
- Для уголка равнополочного: `<side>x<side>x<thickness>`. Пример: `50x50x5`.
- Для уголка неравнополочного: `<big>x<small>x<thickness>`. Пример: `75x50x6`.
- Для швеллера: `<номер>П` или `<номер>У`. Пример: `10П`, `20У`.
- Для двутавра: `<номер>Б1`, `20Ш1`. Пример: `20Б1`.
- Для листа: `<length>x<width>x<thickness>`. Пример: `1500x3000x2`.

## Вопросы, которые закрываем позже

1. **Полный сортамент заказчика** — у него есть свой рабочий список типоразмеров? Его берём как стартовое содержимое JSON.
2. **Нестандартные марки** — использует ли заказчик марки вне основного списка?
3. **Метизы и крепёж** — отдельный каталог или не включаем в MVP?
4. **Сварные швы** — моделируем как компоненты или нет? Если да — отдельный каталог типов швов.
5. **Импорт из XLSX/CSV** — многие таблицы сортамента гуляют в Excel, иметь конвертер из xlsx → json было бы полезно.
