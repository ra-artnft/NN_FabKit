# catalog/ — справочники сортамента

JSON-файлы со структурированными данными о металлопрокате и марках стали. Используются параметрическим генератором геометрии и экспортёрами.

Схема формата описана в `../06-sortament-ontology.md`.

## Текущее содержимое

| Файл | Покрытие | Стадия |
|---|---|---|
| `gost-30245-rect-tube.json` | Профильная труба прямоугольная и квадратная, 28 типоразмеров от 15x15x1.5 до 180x180x6 | Первоначальный набор, расширяется по потребностям |
| `gost-19903-hot-sheet.json` | Лист горячекатаный, толщины 1.5–20 мм + стандартные размеры стока | Первоначальный набор |
| `steel-grades.json` | 10 марок (Ст3сп, Ст3пс, Ст3кп, 09Г2С, С345, С245, 20, 45, AISI 304, AISI 316) | Базовый список, дополняется под реальные используемые марки |

## Правила ведения

1. **Один стандарт — один файл.** Не смешиваем 30245 и 8645 в одном JSON. Это облегчает и ручные правки, и понимание, «откуда взялась цифра».

2. **Каждая запись ссылается на источник.** Поле `source_urls` в корне файла указывает на тексты ГОСТов (docs.cntd.ru, stroyinf.ru, meganorm.ru). При сомнениях — всегда сверяемся по оригиналу стандарта.

3. **`params` — это ИСХОДНЫЕ параметры** (ширина, высота, толщина стенки, радиус). Они нужны геометрическому генератору для корректной отрисовки.

4. **`derived` — это РАССЧИТАННЫЕ величины** (масса, площадь сечения, периметр, момент инерции). Они справочные, используются в спецификациях и расчётах. При исправлении `params` нужно пересчитать `derived`.

5. **Канонические `typesize`** — см. правила в `06-sortament-ontology.md`. Единообразие критично для поиска.

6. **Массы погонного метра** — с точностью до 0.01 кг. Для трубы рассчитывается как `площадь_сечения × плотность × 10⁻⁴`, где площадь сечения — это разница между внешним и внутренним прямоугольниками с учётом радиусов скругления.

7. **Когда записей много** — допустимо делить файл по толщинам (`gost-30245-rect-tube-thin.json`, `*-medium.json`, `*-thick.json`), но это редко нужно — даже полный 30245 (150+ позиций) помещается в один файл размером <50 КБ.

8. **Валидация.** При загрузке плагин проверяет:
   - `$schema_version` поддерживается
   - все items имеют уникальный `typesize`
   - `params` содержит обязательные ключи для данного `profile_type`
   - при failed-валидации — fail-fast с понятной ошибкой в Ruby Console

## Версионирование

Текущая схема — `1.0`. При несовместимых изменениях формата меняем мажорную версию. При добавлении новых опциональных полей — минорную. Старые плагины должны уметь читать новые JSON, игнорируя неизвестные поля.
