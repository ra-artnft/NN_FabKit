# База знаний проекта «Интеграция Claude ↔ SketchUp / LayOut»

## Зачем эта папка

Это Project knowledge для Claude. Файлы здесь — постоянный контекст, к которому Claude обращается в каждом сеансе работы над продуктом: при разборе DC, при написании Ruby-кода плагина, при проектировании экспорта в LayOut и NC-форматы, при интеграции с OpenCutList.

Не обсуждение, не дневник, не черновики. Справочник и карта решений. Писать сюда — значит закрепить знание, которое должно переживать конкретный чат.

## Доменный словарь (минимум для ориентации)

| Термин | Значение |
|---|---|
| **Заказчик** | Основной пользователь-0, для которого делается продукт. Мебельщик + металлоконструкции. Пользуется SketchUp, LayOut, OpenCutList. |
| **DC** | Dynamic Component — встроенный механизм параметризации компонентов SketchUp (родной, не наш). |
| **OCL** | OpenCutList — сторонний GPLv3-плагин для раскроя/спецификации. Используется заказчиком сейчас, в целевой картине заменяется функциями нашего продукта (см. ADR-016). |
| **NC-экспортёр** | Модуль продукта, превращающий SketchUp-модель в программу ЧПУ (IGES, DXF, др.). |
| **Сортамент** | Каталог металлопроката с параметрами (размеры, радиусы, массы, ГОСТ). |
| **Параметрический генератор** | Ядро плагина. Рисует корректную геометрию компонента по параметрам из сортамента. |
| **Наш attribute_dictionary** | `nn_fabkit` — словарь, куда пишем метаданные (марка стали, ГОСТ, NC-флаги). Параллельно DC, не мешает сторонним плагинам. |
| **Referenced plugin** | `mhyrr/sketchup-mcp` (MIT) — форкаем, расширяем LayOut и сортамент. |

## Правила чтения/обновления

- При новом чате Claude первым делом просматривает `09-architecture-decisions.md`, чтобы понимать зафиксированные решения.
- Конкретные технические вопросы — смотри 01–07.
- Референсные образцы заказчика — в 08.
- Все изменения API/стандартов логируй в соответствующем файле + добавь запись в `architecture-decisions.md`, если это меняет решение.
- Язык файлов — русский. Термины API (`Sketchup::Model`, `start_operation`, и т.д.) — в оригинале на английском.
- Код-примеры всегда рабочие (копипастимые), без псевдокода.
