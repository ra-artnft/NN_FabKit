# corpus

Корпус реальных и учебных заказов заказчика-0. Используется для квази-обучения Claude (Project Knowledge / MCP, см. [ADR-015](../docs/knowledge-base/09-architecture-decisions.md)) — Claude видит структурированные примеры и работает «как если бы их знал»: правильные паттерны, узнаваемый стиль заказчика, tacit knowledge.

## Структура

```
corpus/
├── README.md                                  # этот файл (в git)
└── examples/                                  # ⚠ исключено из git (PII клиентов)
    └── <NN-короткое-имя>/
        ├── model.dump.json                    # обязательно — выгрузка через skp_dump
        ├── notes.md                           # обязательно — ключевой артефакт (см. шаблон)
        ├── model.skp                          # опционально — копия рабочего файла
        ├── model.preview.png                  # опционально — общий вид 1024×768
        ├── tz-svarshchiku.md                  # опционально — ТЗ для исполнителя
        ├── tz-svarshchiku.pdf                 # опционально
        ├── nc/                                # опционально — *.igs, *.dxf на станок
        └── photos/                            # опционально
```

Шаблон `notes.md` — [`docs/knowledge-base/templates/example-notes.template.md`](../docs/knowledge-base/templates/example-notes.template.md). Структура папки и `notes.md` — канон по [ADR-015](../docs/knowledge-base/09-architecture-decisions.md).

## Правила приватности

`corpus/examples/` целиком исключён из git (`.gitignore`). Причина: внутри `model.dump.json` в поле `model.path` сохраняются полные пути с именами клиентов (`Yandex.Disk\<Имя Фамилия>\...`). В `notes.md` тоже фигурируют имена.

Если корпус (или его подмножество) понадобится опубликовать — сначала **обезличить**: пути в `model.path`, имена и ссылки в `notes.md`. Опубликование — отдельная процедура с ревью.

## Темп пополнения

ADR-015: **5–8 примеров разного типа** — минимум полезного для MVP. Сейчас 4 (все — перегородки/перила, металл). Не хватает разнообразия:
- что-то преимущественно из листа (короб, кожух)
- что-то с уголком или швеллером
- мелкий быстрый заказ (полка, кронштейн)

> 20+ примеров — уже избыточно для MVP; важнее качество `notes.md`, чем количество папок.

## Процесс пополнения нового примера

1. Заказчик-0 завершает реальный заказ.
2. В SketchUp открывает итоговую модель → `Extensions → NN FabKit → Dump в JSON…` → получает `<basename>.dump.json` рядом с `.skp`.
3. Создаёт папку `corpus/examples/<NN-короткое-имя>/`, переименовывает дамп в `model.dump.json`.
4. Копирует [шаблон](../docs/knowledge-base/templates/example-notes.template.md) в `notes.md`, заполняет.
5. Опционально — кладёт `model.skp`, `model.preview.png`, `tz-svarshchiku.md`, `nc/`, `photos/`.
6. Обновляет таблицу «Текущее наполнение» в этом файле.
7. В следующей сессии Claude увидит новый пример (через MCP / Project Knowledge — по схеме на момент пополнения).
