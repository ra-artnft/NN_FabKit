# MebelFab — заглушка под будущую мебельную ветку

> Предметная ветка: **корпусная мебель** (ЛДСП, фанера, кромка, метизы).
> Namespace: `NN::MebelFab` (планируется).
> Attribute dictionary: `nn_mebelfab` (планируется).
> Аналог `NN::MetalFab` для металлической ветки. См. ADR-013.

Подсистемы внутри:

- [skp/](skp/README.md) — MebelFab × SketchUp: генерация листовых деталей, кромка, фурнитура.
- [layout/](layout/README.md) — MebelFab × LayOut: чертежи мебельных проектов, cut-list через OCL Cutting Diagrams.

**Текущее состояние**: пусто. Заказчик-0 ещё не дал вводных по мебельной ветке (всё внимание на металл — труба, лист, NC через IGES). Когда появятся — появится и Notion sub-INDEX, и spec-документы в [docs/specs/](../../../../docs/specs/).

**Когда сюда добавлять код**: запрос пользователя «в MebelFab …» / «мебельная сторона» / «корпус из ЛДСП» / «раскрой плиты» / «кромка / фурнитура».

## Что НЕ нужно делать

- Не путать namespaces: `NN::MetalFab` и `NN::MebelFab` независимы. Общая оболочка — `NN::FabKit`.
- Не импортировать код OpenCutList (GPLv3, см. ADR-016 superseding ADR-003). Интеграция только через канонические имена материалов.
- Не дублировать сюда металлическую логику (профили, IGES) — она в [../metalfab/](../metalfab/).
